# 页面设计说明（桌面优先）

## 全站通用规范

### Layout
- 桌面端（≥1200px）：内容容器宽度 1120–1200px 居中；页面采用“头部导航 + 主内容分区 + 页脚”三段式。
- 平板（768–1199px）：容器自适应 92% 宽；网格从 3/4 列降为 2 列。
- 移动端（<768px）：单列堆叠；导航折叠为汉堡菜单；卡片间距增大以利触控。
- 主要布局方式：Flexbox（导航/对齐）+ CSS Grid（卡片列表/图片墙）。

### Meta Information（默认值，可被各页覆盖）
- title：{公司名}｜{页面名}
- description：一句话概述公司与核心业务
- Open Graph：og:title / og:description / og:image（默认使用首页首屏图或 Logo）

### Global Styles（设计 Tokens）
- 背景色：#FFFFFF；分区浅底：#F7F9FC
- 主色：#0B5FFF（可信/科技感）；强调色：#FF7A00（CTA）
- 文本色：#111827；次级文本：#6B7280；分割线：#E5E7EB
- 字体层级：H1 40/48，H2 28/36，H3 20/28，Body 16/24，Caption 12/18
- 按钮：
  - Primary：主色底白字；hover 加深 8%；disabled 降低透明度
  - Secondary：白底主色描边；hover 浅底
- 链接：主色；hover 下划线
- 动效：仅使用轻量过渡（150–200ms），避免花哨影响“企业可信度”

### 组件（全站复用）
- 顶部导航（Header）：Logo、一级菜单（首页/关于/业务/案例/新闻/联系）、右侧主 CTA（“立即咨询”跳转联系页）
- 页脚（Footer）：公司简称、电话/邮箱/地址、ICP备案（可预留）、版权信息
- 内容区块（Section）：标题+副标题+内容；可选右上“更多”链接
- 卡片（Card）：图片/标题/摘要/标签；统一阴影与圆角（8px）

## 1) 首页

### Meta
- title：{公司名}｜官网首页
- description：展示公司定位、核心业务、精选案例与最新动态

### Page Structure
- 纵向堆叠的多分区 Landing Page：Hero → 业务入口 → 优势亮点 → 精选案例 → 新闻摘要 → 联系引导。

### Sections & Components
1. Hero 首屏
   - 左侧：公司定位标题（H1）、一句话价值主张（2 行内）、2 个按钮（主：立即咨询；次：查看业务）
   - 右侧：品牌主视觉图（可为工程/办公/产品场景图）
2. 业务入口（Grid 卡片）
   - 3–6 张业务卡：图标/标题/一句话描述；点击进入“业务服务”并定位
3. 优势亮点（图标列表）
   - 3–6 个优势点：如“响应快/交付稳/合规资质/售后保障”等（由后台配置）
4. 精选案例（卡片 + 轮播可选）
   - 展示 3–6 条；卡片点击进入案例详情
5. 新闻摘要（列表）
   - 展示最新 3–5 条：标题+日期+摘要；点击进入新闻详情
6. 联系引导（CTA 区）
   - 大按钮跳转联系页；旁边展示电话与工作时间

## 2) 关于我们

### Meta
- title：{公司名}｜关于我们
- description：公司简介、资质与核心优势

### Page Structure
- 上：页面标题区（Banner，可轻量）
- 中：简介/优势/资质分区

### Sections & Components
1. 简介区
   - 左文右图：公司简介（富文本）+ 形象图
2. 核心优势
   - 图标 + 文案的 2/3 列网格
3. 资质/证照
   - 图片墙（Grid）；点击放大预览（Lightbox）

## 3) 业务服务

### Meta
- title：{公司名}｜业务服务
- description：业务清单与服务能力说明

### Page Structure
- 左侧（桌面）：业务目录列表（sticky）
- 右侧：对应业务详情（锚点分段）

### Sections & Components
1. 业务目录（Sidebar）
   - 列表：业务名称；点击滚动到详情锚点
2. 业务详情（主内容）
   - 每个业务：标题、摘要、适用场景、交付内容、流程/周期（可选）
   - 业务内 CTA：引导到联系页并带上“咨询主题=该业务”（可通过 URL 参数）

## 4) 案例

### Meta
- title：{公司名}｜案例
- description：展示落地项目与成果

### Page Structure
- 顶部筛选条 + 下方案例卡片网格

### Sections & Components
1. 筛选/标签
   - 标签 chips：行业/服务类型（来自后台 tags）；选中后过滤
2. 案例列表（Grid）
   - 卡片：封面、标题、行业/地点、摘要
3. 案例详情（/cases/:id）
   - 顶部：封面图 + 标题信息（行业/地点/时间）
   - 正文：背景-方案-结果（结构化段落）
   - 图集：网格图片；点击放大
   - 底部 CTA：立即咨询（跳转联系页）

## 5) 新闻

### Meta
- title：{公司名}｜新闻
- description：公司动态与行业资讯

### Page Structure
- 列表页：时间倒序的新闻列表

### Sections & Components
1. 新闻列表（List）
   - 单条：标题、日期、摘要；点击进入详情
2. 新闻详情（/news/:id）
   - 标题区：标题、发布时间、分类（可选）
   - 正文区：富文本（支持图文混排）
   - 导航：上一条/下一条

## 6) 联系我们

### Meta
- title：{公司名}｜联系我们
- description：联系信息与在线咨询表单

### Page Structure
- 左侧联系信息 + 右侧表单；下方地图（或右侧地图）

### Sections & Components
1. 联系信息卡
   - 电话（tel）、邮箱（mailto）、地址（外链地图）、工作时间
2. 在线咨询表单
   - 字段：姓名、电话/邮箱（二选一必填）、咨询主题、内容（必填）
   - 提交后：成功提示；错误校验（必填/格式）
3. 地图
   - 优先方案：静态地图图 + “打开地图”外链按钮（加载更快）

## 8) 后台管理（/admin）

### Meta
- title：后台管理｜{公司名}
- robots：noindex

### Page Structure
- 左侧侧边栏导航 + 右侧内容区（Dashboard 结构）

### Sections & Components
1. 侧边栏
   - 菜单：站点信息、关于、业务、案例、新闻、线索、媒体素材
2. 内容列表页（通用）
   - 表格/列表：标题、状态（已发布/草稿）、更新时间
   - 操作：新建、编辑、发布/下线、删除（删除可二次确认）
3. 内容编辑页（通用）
   - 基础字段：标题、摘要、封面、标签、正文（富文本或结构化编辑器）
   - 预览：在右侧或新窗口预览前台效果
4. 媒体素材
   - 上传图片、复制链接、按时间排序
5. 线索管理
   - 列表：姓名、联系方式、主题、时间、状态
   - 操作：标记已读/处理中/完成（可精简为已读/未读）
