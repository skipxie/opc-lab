# 页面设计文档（桌面优先）

## 全局设计规范（适用于所有页面）

### Layout
- 桌面优先：基准宽度 1200px 内容容器（max-width: 1200px；左右 padding 24px）。
- 页面结构：顶部固定导航（sticky），主体内容为分段堆叠（stacked sections），关键区域用卡片网格（CSS Grid）。
- 响应式：
  - ≥1200px：两栏布局优先（例如政策地图）。
  - 768–1199px：两栏可降级为上下布局，侧栏折叠。
  - <768px：导航变为抽屉菜单；列表与地图分 Tab。

### Meta Information（默认）
- title 模板：`单人成军｜AI 驱动的一人公司增长与政策地图`
- description：`用 AI 把目标拆成行动清单；用政策地图找到你能用的资源；加入社区获得同伴与复盘。`
- Open Graph：
  - og:title 同 title
  - og:description 同 description
  - og:type: website

### Global Styles
- 主题：浅色为主，强调“清晰、行动、可信”。
- 色彩 Token：
  - --bg: #0B1020（深色分区背景，可用于首屏/脚注）
  - --surface: #FFFFFF
  - --text: #0F172A
  - --muted: #64748B
  - --primary: #2563EB（行动蓝）
  - --accent: #22C55E（可用/通过状态）
  - --danger: #EF4444
  - --border: #E2E8F0
- 字体与层级：
  - H1 40/48，H2 28/36，H3 20/28，Body 16/24，Caption 12/16。
- 按钮：
  - Primary：背景 --primary，hover 加深，disabled 降低不透明度。
  - Secondary：白底描边 --border，hover 背景 #F8FAFC。
- 链接：默认 --primary，下划线仅在 hover 显示。
- 动效：仅使用轻量过渡（150–200ms）用于 hover、抽屉出现、筛选面板展开。

### 共用组件
- TopNav：Logo（左）、主导航（首页/政策地图/社区加入）、右侧主按钮“加入社区”。
- Footer：联系方式、信息来源说明、隐私说明、版权。
- ContentCard：用于政策卡/案例卡/权益卡。
- Toast：用于“已复制/已收藏/提交成功”。

## 1) 首页（/）

### Layout
- 主体为纵向分段堆叠；关键转化区使用两列（文案+示例/图片）。
- 采用 CSS Grid：精选政策区 `grid-template-columns: repeat(3, 1fr)`（桌面），平板变 2 列，手机 1 列。

### Meta Information
- title：`单人成军｜AI + 政策地图，让一人公司跑起来`
- og:image：站点统一封面图（建议 1200×630，深底+主张文案）。

### Page Structure
1. Sticky 顶部导航
2. Hero 首屏（深底）
3. “AI 如何帮你行动”区
4. “政策地图”预览区
5. “社区加入”转化区
6. 可信度与声明区
7. Footer

### Sections & Components
- Hero（首屏）
  - 左侧：
    - H1：`AI 驱动的单人成军系统`
    - 副标题：`从目标到行动清单；从政策到资源落地。`
    - 两个 CTA：Primary“打开政策地图”，Secondary“加入社区”
  - 右侧：示例卡（静态）
    - 输入：`我想在 30 天内启动一个小产品`（展示为输入框样式）
    - 输出：3–5 条行动清单（卡片列表）
- AI 能力区（叙事 + 结构化）
  - 三步法组件（Stepper）：`收集信息 → 拆解任务 → 每周复盘`
  - 每步配 1 句解释 + 1 个“可交付物”（如：模板/清单/复盘表）
- 政策地图预览区
  - 顶部：标题 + 说明（信息来源与时效性）
  - 内容：精选政策卡 3–6 张
    - 卡片字段：标题、地区标签、类型标签、截止日期（若有）、“查看详情”
  - 右侧（或底部）：筛选快捷 chips（补贴/税务/人才/融资）点击后跳转到 `/policy-map?tag=...`
- 社区加入转化区
  - 左：权益列表（3–5 条）
  - 右：加入提示卡（响应时间、适合人群、隐私提示）+ Primary CTA
- 可信度区
  - 三列信息卡：`我们是谁`、`信息如何更新`、`你将获得什么`
  - 合规小字：不构成法律/财税建议，链接到政策原文

## 2) 政策地图（/policy-map）

### Layout
- 桌面两栏：左侧地图（flex: 1），右侧列表/筛选（宽 420px）。
- 右侧采用“上：筛选面板，下：结果列表”，列表可滚动（sticky header）。
- 地图与列表联动：点击地图点 → 打开右侧详情抽屉；点击列表项 → 地图居中并高亮。

### Meta Information
- title：`政策地图｜按地区快速找到可用政策`
- description：`筛选你所在地区与关注方向，查看政策要点与官方链接。`

### Page Structure
1. 顶部导航（复用）
2. 工具条（筛选、搜索、分享当前筛选）
3. 地图 + 结果侧栏
4. 政策详情抽屉（右侧覆盖层）

### Sections & Components
- 工具条（Toolbar）
  - 搜索框：占位符 `搜索政策标题/关键词`
  - 筛选按钮：打开筛选面板（桌面默认展开，移动端抽屉）
  - 按钮：`复制当前筛选链接`
- 筛选面板（Filters）
  - 地区选择：省/市下拉（可先做城市文本输入 + 建议列表）
  - 类型：多选 chips（补贴/税务/场地/人才/融资/其他）
  - 人群：多选（个体户/一人公司/初创团队/科技型企业）
  - 更新时间：最近 30/90 天
  - 操作：清除、应用
- 地图（Map）
  - 点位：按地区聚合（cluster）
  - 点位状态色：有截止日期且临近 → 提醒色（--danger）；长期有效 → --accent
  - 空态：无匹配结果时提示“换个筛选试试”
- 结果列表（Results List）
  - 列表项：标题（两行截断）、地区、类型、更新时间
  - 操作：收藏（本地）、查看详情
- 政策详情抽屉（Detail Drawer）
  - 结构：标题 + 标签 + 关键字段
    - 要点摘要、适用条件、材料清单、截止时间、来源与官方链接
  - 操作：复制摘要、分享、打开官方链接（新窗口）
- 线索提交入口（轻量）
  - 在列表底部或详情抽屉末尾：`发现了新政策/更新？提交线索`
  - 触发表单弹窗：标题、链接、补充说明、联系方式（可选）

## 3) 社区加入（/community）

### Layout
- 顶部叙事 + 权益网格 + 表单两栏（桌面）。
- 表单为右侧固定卡片（sticky），左侧为说明内容滚动。

### Meta Information
- title：`加入社区｜和同伴一起单人成军`
- description：`填写信息后我会与你联系确认；你将获得每周政策解读与 AI 工作流模板。`

### Page Structure
1. 顶部导航（复用）
2. 社区主张区（标题 + 适合/不适合）
3. 权益区（卡片网格）
4. 加入流程区（3 步）
5. 报名表单（核心）
6. FAQ + 隐私说明

### Sections & Components
- 主张区
  - H1：`把“一人公司”做成可持续系统`
  - 两列清单：`适合你（3–5 条）` / `不适合你（2–3 条）`
- 权益卡片（Benefits Grid）
  - 卡片：`每周政策雷达`、`AI 行动模板`、`同伴互助`、`复盘机制`
  - 每张卡：一句话收益 + 一个具体交付物例子
- 加入流程（Stepper）
  - Step1 填写信息
  - Step2 收到确认
  - Step3 进入社区开始每周节奏
- 报名表单（Form Card）
  - 字段：称呼（可选）、联系方式类型（邮箱/微信单选）、联系方式值（必填）、所在城市（可选）、当前阶段（下拉）、关注方向（多选）、隐私同意（必选）
  - 提交按钮：Primary“提交申请”
  - 成功态：显示“已收到，我会在 X 小时内联系你”+ 返回政策地图按钮
  - 错误态：字段级错误提示（红色说明）
- FAQ
  - 折叠面板（Accordion）：费用、内容频率、隐私、是否广告、是否适合新手
- 隐私说明
  - 简短声明：仅用于联系与服务改进；提供删除请求联系方式
